## PHASE 2: NEW ROUTES CREATED

### 2.1 Timetable Routes

**File**: `routes/timetable.js` (NEW)
**Endpoints Created**:
1. `GET /api/timetable/:semester?section=A`
   - Returns timetable for specific semester + section
   - Example: GET /api/timetable/Semester%201?section=A
   - Response: Full schedule with time slots, classes, faculty, rooms

2. `GET /api/timetable/faculty/:facultyId`
   - Returns all classes assigned to a faculty member
   - Example: GET /api/timetable/faculty/10001
   - Response: Array of all faculty's classes across all semesters

3. `GET /api/timetable?facultyId=10001`
   - Returns faculty's complete schedule structure
   - Used for faculty dashboard
   - Response: Nested schedule by semester|section

**Database Integration**:
- Reads from `Timetable` collection in MongoDB
- One document per (semester, section) combination
- 12 timetable records (4 semesters × 3 sections)
- Generated by `seed.js` with realistic class assignments

**Data Flow**:
Backend: seed.js → MongoDB Timetable collection → API routes → Frontend
- NO hardcoded timetable anywhere
- Students see timetable from their semester/section
- Faculty see all their assigned classes

---

### 2.2 Reminders Routes

**File**: `routes/reminders.js` (NEW)
**Endpoints Created**:
1. `GET /api/reminders?facultyId=10001`
   - List all reminders for faculty (sorted by date)
   - Response: Array of reminder objects

2. `GET /api/reminders/:id`
   - Get single reminder by MongoDB _id

3. `POST /api/reminders`
   - Create new reminder
   - Required: title, date, description, facultyId
   - Returns: Created reminder object

4. `PUT /api/reminders/:id`
   - Update reminder details
   - Can update: title, date, description

5. `DELETE /api/reminders/:id`
   - Delete reminder from database
   - Faculty dashboard reflects changes immediately

**Database Integration**:
- Writes to/reads from `Reminder` collection
- Indexed by facultyId for fast lookup
- Initial seed data: 5 sample reminders
- Faculty adding reminders: Real-time DB updates

**Data Flow**:
Backend: API routes ↔ MongoDB Reminder collection ↔ Frontend
- Faculty Dashboard polls/refreshes reminders
- No client-side caching (cache invalidated on save/delete)
- Real-time state from database
